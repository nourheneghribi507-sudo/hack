# TPL – AI-powered Club Management System

L is a high-fidelity, hackathon-ready mobile application prototype designed for student club management. It leverages a multi-agent AI system to streamline events, communication, and security for student organizations.

## 🚀 Key Features

### 1. Multi-Agent AI System
TPL features four specialized AI agents that collaborate to manage the club:
- **Architect Agent**: Specialized in event planning. Suggests dates, venues, event timelines, and budget allocations based on historical data.
- **Liaison Agent**: Handles external communications. Generates social media posts, sponsorship emails, and marketing drafts.
- **Archivist Agent**: Manages the club's knowledge base. Retrieves documents from the secure vault and analyzes past data for insights.
- **Sentinel Agent**: The security backbone. Monitors system logs, detects suspicious activity, and provides real-time security alerts.

### 2. Secure Document Vault
A simulated high-security storage for sensitive club documents (sponsorship decks, budgets, member lists), featuring a cyberpunk UI with glassmorphism and real-time search.

### 3. Event Management
A dedicated hub to track upcoming activities, including "AI Suggestions" from the Architect agent directly integrated into event cards.

### 4. Interactive AI Chat
A seamless chat interface where users can switch between agents to get specialized help. Styled with vibrant neon glows and smooth transitions.

## 🎨 Design Philosophy
- **Cyberpunk Aesthetic**: Dark mode TPL by default (`#020617`), using neon cyan and magenta accents for high contrast and a futuristic feel.
- **Glassmorphism**: Extensive use of background blurs and semi-transparent cards to create depth.
- **Mobile-First**: Fully responsive design optimized for mobile viewports, following the provided Figma UX patterns.

## 🛠️ Tech Stack
- **Frontend**: Vanilla HTML5, CSS3 (with advanced gradients/animations), and JavaScript (ES6+).
- **Icons**: Lucide Icons.
- **Typography**: Inter & Orbitron (Google Fonts).
- **AI Simulation**: Custom state-controlled response engine for the hackathon prototype.

## 📂 Project Structure
- `index.html`: The main application shell and screen layouts.
- `index.css`: Core design system, cyberpunk styles, and animations.
- `app.js`: Application logic, navigation, and AI agent simulations.
- `assets/`: Custom generated branding and avatar assets.
